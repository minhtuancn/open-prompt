# Open Prompt

Tauri v2 AI overlay — global hotkey (Ctrl+Space) → React 18 overlay → Go sidecar engine → multi-provider AI.

## Architecture

```
Tauri v2 (Rust)  ←→  React 18 + Zustand  (src/)
       ↓ spawns
Go Engine sidecar (go-engine/)  ←→  SQLite ~/.open-prompt/open-prompt.db
       ↓ HTTPS
Anthropic / OpenAI / Gemini / Copilot / Ollama
```

IPC: JSON-RPC 2.0 over Unix socket (Linux/macOS) or TCP (Windows).
Every request is wrapped in `{ "secret": "<OP_SOCKET_SECRET>", "request": {...} }`.

## Commands

### Go Engine
```bash
cd go-engine
go build ./...          # build check
go test ./... -race     # run all tests
make build-linux        # cross-compile → bin/go-engine-linux-amd64
```

### Frontend
```bash
npm run build           # tsc + vite build → dist/
npm run tauri dev       # full app in dev mode
```

### Cross-platform sidecar (before Tauri build)
```bash
./scripts/build-engine.sh   # builds Go for all platforms → src-tauri/binaries/
```

## Key Files

| File | Purpose |
|------|---------|
| `go-engine/main.go` | Sidecar entry — reads OP_SOCKET_SECRET, opens DB, listens |
| `go-engine/api/router.go` | All RPC handlers wired up here |
| `go-engine/api/handlers_query.go` | Streaming AI query handler |
| `go-engine/api/types.go` | Request / Response / RPCError structs |
| `src-tauri/src/ipc.rs` | `call_engine` Tauri command — routes socket vs TCP |
| `src-tauri/src/sidecar.rs` | Spawns Go engine, generates per-session secret |
| `src/hooks/useEngine.ts` | `callEngine()` + `streamQuery()` — React RPC client |
| `src/App.tsx` | Root component — state machine (loading→onboarding→login→overlay→settings) |

## Non-Obvious Patterns

**Streaming flow** — listen *before* calling, not after:
```typescript
const unlisten = await listen('stream-chunk', handler)  // set up listener first
await callEngine('query.stream', params)                 // then trigger
```
Go engine sends `stream.chunk` Notifications (no `id` field) line by line. Frontend accumulates chunks via `overlayStore.appendChunk()`.

**Request envelope** — Tauri wraps every RPC call with a secret:
```json
{ "secret": "64-hex-chars", "request": { "jsonrpc": "2.0", "method": "...", "params": {...}, "id": 1 } }
```
Secret is generated fresh each app launch (32 random bytes → hex). JWT secret is derived from it: `HMAC-SHA256(socket_secret, "jwt-signing-key")`.

**Go handler signature**:
```go
func (r *Router) handleFoo(req *Request) (interface{}, *RPCError)
// Use decodeParams(req.Params, &p) to extract typed params
```
Streaming handlers take an extra `conn net.Conn` as first arg (see `handleQueryStream`).

**Repo pattern**:
```go
repo := repos.NewFooRepo(db)
// CRUD: Create, FindByID, List(userID, limit), Update, Delete
// All List methods cap at limit=1000; use clampLimit() helper
```

**DB**: Pure Go SQLite (no cgo). Opens `~/.open-prompt/open-prompt.db`. Migrated at startup (idempotent `CREATE TABLE IF NOT EXISTS`). WAL mode, 1 connection. Override with `OP_DB_PATH`.

**Windows** uses TCP instead of Unix socket — Go engine prints `ready:<addr>` to stdout, Tauri reads and stores the port. No code changes needed; `ipc.rs` handles both paths.

**Input validation helpers** (in `go-engine/api/validation.go`): `truncateString(s, maxLen)`, `clampLimit(limit, defaultVal)`, `extractToken(req)` — use these in new handlers instead of inline logic.

## Environment Variables

| Variable | Default | Purpose |
|----------|---------|---------|
| `OP_SOCKET_SECRET` | (generated by Tauri at launch) | Shared secret for socket auth — required |
| `OP_SOCKET_PATH` | `/tmp/open-prompt.sock` | Override Unix socket path |
| `OP_DB_PATH` | `~/.open-prompt/open-prompt.db` | Override database path |

## Testing

```bash
# Go — integration tests use real SQLite (in-memory)
cd go-engine && go test ./... -race

# Frontend — TypeScript type check only (no JS test runner configured)
npx tsc --noEmit
```

Test helpers: `db.OpenInMemory()` for in-memory SQLite, `newTestDB(t)` in `repos_test` package seeds a user and migrates schema.

## Zustand Stores

| Store | Persistence | Key State |
|-------|-------------|-----------|
| `authStore` | localStorage | token, username, userId |
| `overlayStore` | none (session) | input, chunks, isStreaming, activeProvider |
| `settingsStore` | localStorage | fontSize, theme, language |
| `providerStore` | none | providers list (cached per token via `hasFetched` flag) |

## Rate Limiting

Keyed by JWT user ID (`user:<id>`) when authenticated, falls back to remote address.
Only 4 methods are limited: `auth.login` (5/min), `auth.register` (3/min), `marketplace.publish` (5/min), `marketplace.search` (30/min).
